01 / Overview

What We're Building

AgFinTax Planning AI is not a chatbot that regurgitates tax tips. It is a structured, intake-driven planning engine that profiles a client's full financial picture — via voice, document upload, or guided Q&A — and maps them to 57 specific, actionable tax strategies with real dollar estimates, IRC citations, and implementation roadmaps.

Think of it as the brain of a senior tax advisor who knows every strategy in the Corvée 2024 Tax Strategies Masterclass, has memorized the 2025 tax code, and never forgets to ask the right qualifying question.

Key Design Decisions

Low Temperature (0.1–0.3)
Tax advice demands accuracy, not creativity. Chat uses 0.3 for slight warmth; extraction and classification use 0.1 for near-deterministic output.
3-Model Failover Chain
Claude Sonnet 4 → Claude 3.5 Sonnet → GPT-4o Mini. Each model gets a 45-second timeout. Users never see "AI unavailable" errors.
JSON Repair Logic
LLMs sometimes output malformed JSON. The plan generator strips markdown fences, removes trailing commas, auto-closes unclosed structures, and validates required fields.
Entity-Specific Prompts
An S-Corp owner gets prompted about reasonable compensation and FICA savings. A C-Corp gets QSBS exclusion and retained earnings. Never generic advice.

The voice system isn't bolt-on dictation. It's a full two-way conversation where the AI speaks questions, listens for answers, interprets natural language responses, and advances through a structured qualification flow — with silence detection, muting, skipping, and section transitions.

State Management

The app uses a deliberately simple state architecture — localStorage for persistent data (client profiles, saved plans, documents) and React hooks for transient UI state. No Redux, no Zustand, no global state library. This works because:

Why localStorage?
Tax planning is inherently session-based. A client fills out their profile, gets a plan, maybe comes back to refine it. localStorage gives instant persistence without Supabase round-trips during the intake flow. The plan is the unit of work — save it when it's ready, not on every keystroke.
When Supabase?
Supabase is wired up and ready but currently underutilized. The path forward: save finalized plans to Supabase (linked to Clerk user ID), store document metadata and OCR results, enable multi-device access, and power advisor dashboards.

10 / Gaps & Strategic Opportunities

What Would Make This a 10x Product

The foundation is strong. These are the moves that would take AgFinTax from "impressive demo" to "indispensable tool that tax advisors can't live without."

1. Strategy Interaction Graph
Right now strategies are evaluated independently. In reality, they interact: S-Corp election affects reasonable compensation which affects FICA savings which affects retirement plan contribution limits. Build a dependency graph so the plan reflects these cascading effects and prevents contradictions (e.g., don't recommend both S-Corp election AND hiring children under sole prop rules).
2. OCR → Strategy Auto-Qualification
When a client uploads a 1040, the OCR extracts AGI, deductions, credits used, schedules filed. That data should automatically answer qualification questions. If Schedule C shows $180K income and no retirement contributions, the system should pre-qualify Solo 401(k), SEP IRA, and S-Corp election without asking.
3. Supabase-Backed Persistence + Multi-Device
Move plans, profiles, and conversation history from localStorage to Supabase. This unlocks: resuming sessions across devices, advisor review workflows, plan versioning (before/after comparison), and analytics on which strategies are most recommended.
4. Advisor Dashboard (B2B Play)
Build a separate view for Anil and his team to see all client plans, track implementation progress, assign action items, and get alerts when a client's situation changes. This turns AgFinTax from a client self-service tool into a full practice management layer.
5. Real-Time Savings Calculator with Sliders
Instead of static estimates, let users drag sliders (income, contribution amounts, property values) and watch their tax savings update in real-time. Show the math: "If you contribute $23,500 to 401(k) at your 32% marginal rate, you save $7,520 in federal tax." Make the value tangible and interactive.
6. Deadline & Election Tracker
Tax planning is time-sensitive. S-Corp election (Form 2553) has a March 15 deadline. Estimated payments are quarterly. 1031 exchanges have 45-day and 180-day windows. Build a calendar that tracks every deadline tied to the client's plan, sends reminders, and shows what happens if they miss it.

11 / Suggested Roadmap

Phased Implementation Plan

Based on the gap analysis, here's a phased roadmap that balances quick wins with high-impact features. Each phase builds on the previous one.

Phase 1 — Production Readiness (2–3 weeks)

Get the existing features production-solid before adding new ones.

Supabase persistence for plans & profiles
Move from localStorage to Supabase. Link plans to Clerk user IDs. Enable plan versioning. This is the single most important infrastructure upgrade — everything else depends on it.
Document upload to Supabase Storage
Replace the stub endpoint with real file storage. Store OCR results alongside document metadata. Enable multi-document accumulation per client.
Wire savings dashboard to live plan data
Replace placeholder charts with Recharts visualizations driven by the actual saved plan. Show breakdown by category, entity comparison, and implementation timeline.
Dynamic tax review from profile + plan
Generate red/green flags by comparing the client's uploaded returns against their recommended strategies. "You filed Schedule C but aren't taking the home office deduction — potential $3,200 savings."

Phase 2 — Differentiation (4–6 weeks)

Features that make AgFinTax clearly better than anything else on the market.

OCR → auto-qualification pipeline
When a client uploads their 1040, automatically answer qualification questions based on extracted data. Skip the interview for things the return already tells us. "We see you have Schedule E rental income — we've pre-qualified you for Cost Segregation and REPS evaluation."
Strategy interaction modeling
Build a dependency graph between strategies. Flag conflicts, amplifiers, and prerequisites. Example: "Recommend S-Corp election BEFORE reasonable compensation analysis." Or "Cash Balance Pension works best when combined with 401(k) profit sharing."
Interactive savings calculator with sliders
Let users adjust income, contribution amounts, property values — and watch savings recalculate in real-time. Show the marginal rate math. Make the value visceral.
Scenario comparison engine
"What if I elect S-Corp?" vs "Stay as LLC" — side-by-side with real numbers. Include: FICA savings, additional compliance costs, payroll overhead, net benefit. This is the #1 question every business owner asks.

Phase 3 — Scale & Monetize (2–3 months)

Turn AgFinTax into a platform, not just a tool.

Advisor dashboard & practice management
